Responsive Web Design

A single web site design which dynamically responds (adapts) to the characteristics (screen width, resolution, capabilities) of the display device.

The same web site at the same URL works and provides an appropriate experience across all devices:

  • tablet
  • laptop

Being Responsive

  • Responsive Layout: dynamically adjust the layout to best suit the available window size (width).
  • Responsive Content: dynamically adjust embedded media (text, tables, images, videos) to the available space and display charactersistics (resolution).
  • Responsive Navigation: e.g. collapse menus to a single button.
  • Responsive Interaction: sniff features to selectively add support for location, orientation, tilt, shaking, touch gestures (swipe, pinch, …), etc.

Progressive Enhancement

  1. Build baseline first (HTML only).
  2. Then add styling (CSS).
  3. Then add more interactivity (Javascript).

[as opposed to graceful degradation]

  • Aaron Gustafson; Understanding Progressive Enhancement; A List Apart, 07 Oct 2008. http://alistapart.com/article/understandingprogressiveenhancement

  2. Then expand to other devices.

Breakpoint Diagram

Breakpoints

  • Between Breakpoints: Fluid (Scalable) Layout.
    • Heights in em or rem.
    • Widths in %, em, or rem.
    • Absolute units (px, mm, cm, in) are forbidden!
    • Use vector graphics (SVG) where possible.
    • Flexbox is wonderful!
  • At Breakpoint: Change Layout.
    • Say, from 2-column to 3-column layout.

Specifying Breakpoints with CSS3 Media Queries

/* settings for window-width: narrow */
@media only all and (max-width: 40em) {
...
}

/* settings for window-width: medium */
@media only all and (min-width: 40em) and (max-width: 60em) {
...
}

/* settings for window-width: wide */
@media only all and (min-width: 60em) {
...
}

Content Parity

  • Users expect to have access to the same content and funtionality, no matter how they access the web site.
  • No content should be removed completely in any view.

Responsive Text (Paragraphs)

  • Provide text in different versions (lengths): short, medium, long.
    http://www.frankieroberto.com/responsive_text
    http://mobile.smashingmagazine.com/2012/02/27/ever-justification-for-responsive-text/
  • Limit characters per line for wide displays: 
    max-width: 60em;
    or use CSS3 text wrapping: 
    text-overflow: ellipsis;
    or use CSS3 columns: 
    -webkit-column-count: 2;
-webkit-column-rule: 0.1em solid #bbb;
-webkit-column-gap: 2em;
-moz-column-count: 2;
-moz-column-rule: 0.1em solid #bbb;
-moz-column-gap: 2em;
  • Turn on hyphenation: 
    body {
  hyphens: auto;
  -webkit-hyphens: auto;
  -moz-hyphens: auto;
  -ms-hyphens: auto;
}

code, var, kbd, samp, listing, plaintext, xmp, abbr {
  hyphens: none;
  -webkit-hyphens: none;
  -moz-hyphens: none;
  -ms-hyphens: none;
}
  • Pack content more tightly on narrow screens: 
    line-height: 1.2;
word-spacing: -0.1em;
letter-spacing: -0.05em;
